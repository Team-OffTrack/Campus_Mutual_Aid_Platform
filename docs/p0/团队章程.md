## 一、团队成员与角色分工

### 1. 需求负责人 - 侯乔岳

- **理解：**  
  需求负责人是整个项目需求环节的总负责人，是连接用户、团队与最终产品的关键角色。这个角色不是简单“写文档”，而是要把真实用户的想法变成清晰、可开发、可测试的需求，保证团队做的东西是用户真正需要的，避免方向错误、功能无用或开发返工。同时，在 AI 辅助项目中，需要把控 AI 生成内容的准确性，不被 AI 误导，确保需求真实、合理、可落地。

- **核心职责与目标：**  
  组织用户调研，整理提炼真实互助需求；编写需求规格说明书，完成用例建模与用户故事编写，确保需求清晰可测；输出准确且无矛盾的需求文档，按时高质量完成所有需求相关交付物，减少后期沟通成本。

### 2. 架构负责人 - 杨佳兴

- **理解：**  
  架构负责人是项目技术层面的核心决策者，主要负责从全局视角规划系统结构与技术路线。该角色不仅需要具备扎实的技术功底，还需要具备良好的前瞻性和权衡能力，在复杂需求与现实约束之间做出合理决策。同时，架构负责人还承担着技术方向“定盘星”的作用，确保团队在开发过程中保持一致的技术理念和规范。

- **核心职责与目标：**  
  确保技术方案合理、稳定、可持续演进，为项目长期发展打下基础。构建清晰、可扩展且高性能的系统架构，支持业务持续增长。沉淀技术决策过程，减少重复讨论，提高团队协作效率。

### 3. 开发负责人 - 胡皓轩

- **理解：**  
  开发负责人是代码质量的守门人，核心任务是确保团队产出的代码具有一致性、可读性和可维护性。这个角色需要制定并推行编码规范，通过代码审查发现潜在问题，同时亲自参与核心模块的开发以树立技术标杆。在 AI 协作的背景下，开发负责人还需要特别关注 AI 生成代码的质量审查，防止工具依赖而能力退化现象的发生。开发负责人不仅是代码的编写者，更是团队技术文化的塑造者，需要在效率与质量之间找到平衡点，确保每一行合入仓库的代码都经得起推敲。

- **核心职责与目标：**  
  制定并维护统一的编码规范与代码审查机制，日常管理维护 Git 仓库，确保 AI 生成代码正确标注并保留 Prompt 日志。所有合入代码 100% 经过 Code Review，确保 CI 流水线构建成功率达 95% 以上，核心模块代码覆盖率达标，确保项目在技术层面保持高质量交付。

### 4. 测试负责人 - 沈诺

- **理解：**  
  测试负责人是项目质量的“定海神针”。该角色不仅是在编码完成后进行找错（Bug），更要将质量意识贯穿于项目的全生命周期。从需求评审阶段开始，测试负责人就需介入以确保需求的可验证性；在开发过程中，通过策略制定和环境准备，为高质量交付打下基础。其核心价值在于通过科学的测试方法，最大限度地降低软件发布后的风险，确保交付给用户的产品稳定、可靠。

- **核心职责与目标：**  
  明确测试范围、测试类型以及自动化工具的选择；基于业务逻辑编写覆盖全面的测试用例，并负责执行和记录；监控项目整体质量，为产品是否达到发布标准提供关键依据。确保所有业务功能点均有对应的测试用例覆盖，无盲区交付；通过早期介入识别逻辑漏洞，降低后期修复的成本和风险；确保交付时系统中不存在严重已知缺陷，保障上线后的系统稳定性。

## 二、AI 工具链选型

### 1. AI 编程助手：Gemini

- **选择理由：**  
  对 ChatGPT 和 Gemini 进行评估：
  - 从上下文处理能力看，Gemini 具备超大规模上下文窗口，能够一次性处理整个项目的架构文档、API 定义和历史代码，避免碎片建议导致的架构偏差。
  - 从研发链路融合度看，Gemini 在代码生成、代码审查和自动化单元测试生成方面表现一致，有助于架构规范的有效实施。
  - 从性能与成本平衡看，Gemini 在处理大规模自动化任务时，提供更高的吞吐量和优越的成本效益，符合“务实落地”的原则。

### 2. AI 对话工具：ChatGPT

- **选择理由：**  
  对文心一言和 ChatGPT 进行评估：
  - 从功能上看，ChatGPT 的算力相对更强，支持的语言种类更多，能够更好地作为生成文档、分析需求的辅助 AI 工具。
  - 从易用性和团队熟悉度上看，主要使用者对两个 AI 工具的使用效率、熟悉度相差不大，不作为主要比较内容。
  - 从生成准确性看，ChatGPT 能更好地根据自己的上下文进行应用、迁移，保证了使用中的内容连续性，更适用于完整的文档生成流程。

### 3. AI 代码审查工具：CodeRabbit

- **选择理由：**  
  对 CodeRabbit 和 CodeSpect 进行评估：
  - 从功能上看，CodeRabbit 能够逐行分析代码并给出修改建议，同时提供代码总结和问题提示，功能较为全面且贴合实际开发流程。
  - 从易用性和团队熟悉度上看，CodeRabbit 可直接集成到 GitHub 工作流中，基本无需额外配置，使用门槛较低，更适合初学者团队。
  - 从审查效果与适用性看，CodeRabbit 能够较好识别常见代码问题（如逻辑错误、边界情况缺失等），并结合上下文提供较合理的建议，更适用于本项目这种中小规模开发场景。

### 4. AI 测试工具：Claude

- **选择理由：**  
  对 Claude 和 ChatGPT 进行评估：
  - Claude 的核心优势在于逻辑严密性，它拆解复杂 PRD 时极少遗漏边缘场景，生成的自动化脚本几乎不用改就能跑。
  - 而 ChatGPT 强在多模态和插件生态，分析测试截图或处理海量测试数据更方便。两者都可以免费使用。
  - 但从测试工作的“严谨度”和“代码质量”来看，Claude 目前的性价比和团队上手后的产出质量明显高出一截。

## 三、团队协作与沟通

- **沟通渠道：** 使用 QQ 作为团队日常沟通工具，定期召开站立会和周会同步进度。
- **文档管理：** 所有项目文档存储在 Git 仓库的 `docs/` 目录下，确保版本控制和可追溯性。
- **代码管理：** 使用 Git 进行版本控制，遵循 `main` + `dev` + `feature` 分支策略，确保代码合并的规范性和安全性。

## 四、决策机制

- 重大决策需团队全体成员讨论并达成一致，简单决策由相关负责人决定并同步团队。
