# 校园互助平台 — 交互设计说明

> 配合人机交互课程展示使用。结合课件知识点，说明本项目做了哪些交互设计、为什么这样做。

## 项目简介

一个校园互助 Web 应用。学生可以发布需求（跑腿、交易、组队、失物招领、学习互助），也可以浏览和接取别人的需求，支持私信聊天、双向评价、信誉评分。

**前端**：Vue 3 + Vant 组件库，移动端和桌面端都能用。

---

## 一、可用性方面

> 对应课件第 3 讲：可用性目标六个维度

### 1. 好学吗？（易学性）

首页每个功能都是「图标 + 名称 + 一句话描述」，看一眼就知道是干什么的：

- 🚚 跑腿代取 —— 发布取件需求
- 🛒 二手交易 —— 发布交易信息
- 👥 组队匹配 —— 发布组队需求

填表单的时候，每个输入框都有图标（比如锁=密码）和 placeholder 提示，不用猜。

### 2. 记住了吗？（易记性）

- 导航栏、底部按钮、发布按钮位置所有页面都一样，形成肌肉记忆
- 六种需求类型各有固定配色（跑腿=橙色、交易=绿色、组队=蓝色……），在列表、详情、发布页面都保持一致
- 「我发布的」「我接取的」用 tab 切换，不用去不同页面找

### 3. 功能够用吗？（效用性）

从发布需求 → 浏览搜索 → 接单 → 私信沟通 → 完成确认 → 互相评价 → 信誉积分，整个流程都在平台内完成，不需要切到其他工具。

### 4. 效率高吗？（高效率）

- **手机上**：顶部粘着搜索和排序栏，右下角一个大加号按钮一键发布
- **电脑上**：自动切换成表格视图，信息密度高，一眼扫过去能看很多条
- 首页有快捷入口，一步就能到最常用的功能

### 5. 会误操作吗？（安全性）

- 取消需求、确认接单这些重要操作都会弹确认框，不会手滑搞错
- 提交按钮点一次后就变灰转圈，不会重复提交
- 上传图片会检查是不是图片、有没有超过 5MB
- 管理后台不能封禁自己，有保护

### 6. 好看好用吗？（主观满意度）

- 毛玻璃卡片有层次感，不扁平
- 首页会根据时间显示不同问候语（早上好☀️ / 夜深了🌙）
- 按钮按下去有弹弹的感觉，不是死板的过渡
- Hero 区域用了紫色渐变，视觉上有温度
- 卡片的圆角不完全对称，有些地方大有些地方小，看起来更自然

## 二、设计原则方面

> 对应课件第 3 讲：黄金规则 & 启发式规则

### 一致性——所有页面长得像一个东西

项目把颜色、圆角、阴影、动画、字体全部定义成了 CSS 变量（就跟调色盘一样），放在一个文件里：

```
--c-primary: #6750A4;     // 主色
--r-large: 1rem;          // 大圆角
--s-md: 0 4px 20px ...;   // 中等阴影
--ease: 350ms ...;        // 默认动画
```

所有页面引用这些变量，改一个地方全局生效。不会出现这个页面圆角 10px、那个页面圆角 16px 的情况。

### 状态反馈——让用户知道发生了什么

- **加载时**：显示骨架屏（灰白色占位块有闪光动画），比白屏干等好得多
- **操作完**：弹 toast 提示结果（「发布成功」「已取消」）
- **有未读**：小红点带脉冲动画
- **出错了**：显示错误信息和重试按钮
- **列表为空**：显示空状态图标和文字（「还没有发布过需求」）

### 贴近现实——说人话

- 界面全中文，用学生日常说的词（「跑腿」「组队」「积分」而不是「物流」「匹配」「Point」）
- 时间显示「3 分钟前」「2 小时前」而不是「2026-06-09 14:22:00」
- 图标选的都是日常能见到的（物流车、店铺、朋友、搜索）

### 别让我记——让我认

- 需求类型用彩色标签，不用纯文字。脑子识别颜色比回忆文字快得多
- 状态也标颜色：进行中=绿色、已取消=灰色、已完成=蓝色
- 有头像显示头像，没头像显示彩色首字母

### 别让我犯错——提前挡住

- 表单有校验，空的必填项会提示「请输入标题」
- 重要操作要弹框确认，不是一点就执行
- 提交中按钮禁用，防止连点

### 我想反悔——给退路

- 每个页面左上角有返回箭头
- 需求可以取消
- 评价可以修改
- 封禁错了可以解封

### 简洁——不多放东西

- 卡片上只放必要信息：类型标签 + 标题 + 一句话描述 + 发布者 + 时间
- 留白够多，不挤

## 三、手机和电脑都能用

> 对应课件第 3 讲黄金规则第 2 条「符合普遍可用性」

## 四、动画设计

> 对应课件第 3 讲：EEC 模型中的评估阶段反馈

动画不是为了炫，是为了让用户感知到操作确实生效了：

- **按按钮**：有弹性回弹（像按弹簧），确认操作已被接收
- **页面切换**：轻微上移 + 淡入，过渡自然不突兀
- **装饰元素**：Hero 区域的半透明圆圈缓慢浮动，有空间感但不抢眼
- **未读标记**：脉冲动画吸引注意

用的是什么动画？一种叫「弹簧曲线」的缓动函数，特征是在动画末尾有一点点回弹，模拟真实物理世界的感觉。比如：

```
cubic-bezier(0.27, 1.06, 0.18, 1.00)
```

这个曲线的意思是：动画在结束前会「冲过头」一点点再弹回来，就像弹簧一样。比普通的 ease-in-out 更有生命力。

## 五、迭代设计过程

> 对应课件第 4 讲：形成性评估与迭代设计

看 git 提交记录能看到设计是怎么一步一步改出来的：

1. **先做功能**：需求发布/接取、登录注册、聊天 → 能用
2. **再做美化**：毛玻璃效果、渐变、阴影、动画 → 好看
3. **最后做设计系统**：把所有颜色/圆角/阴影/动画抽成变量 → 好改

第三步的关键是把设计决策变成了「变量」而不是「写死的值」。这样做的好处是：想调整整体风格时，改几个变量就行，不用一个一个文件翻。这就是课件里说的「迭代设计」——评估发现问题 → 低成本修改 → 再评估。

## 六、总结

这个项目在交互设计上做了这些事：

1. **六项可用性目标**都照顾到了：好学好记、功能齐全、效率不错、防手滑、看着舒服
2. **十条启发式规则**大部分都有对应的设计决策：状态可见、贴近现实、一致统一、识别优于记忆、简约、给退路
3. **手机和电脑**各有一套布局，不互相迁就
4. **动画有目的**：反馈操作、引导注意、营造质感，不是单纯炫技
5. **设计系统可迭代**：CSS 变量让改设计变得便宜，支持逐步打磨
